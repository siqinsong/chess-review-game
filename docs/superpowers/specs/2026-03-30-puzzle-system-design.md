# Puzzle 系统优化设计

**日期：** 2026-03-30  
**项目：** chess-review-game  
**范围：** Puzzle 模式题库独立化 + 难度/主题双维度筛选 + 题目选择弹窗

---

## 背景

当前 Puzzle 系统有 6 道题硬编码在 `app.js` 里，无标签/难度/主题字段，只能随机切换，无法筛选。目标是将题库独立为 `puzzles.js`，扩充至 24 道，并提供弹窗式题目选择器。

---

## 数据结构

`puzzles.js` 导出全局变量 `PUZZLES`，每道题结构如下：

```js
{
  id: "rook-vs-king-basic",       // 唯一标识
  title: "车王残局基础",
  difficulty: "beginner",         // beginner | intermediate | expert
  theme: "endgame",               // opening | middlegame | endgame | tactic
  description: "白方先走，用车将死黑王。",
  board: [...],                   // 8×8 数组，同现有格式
  turn: "w",                      // "w" | "b"
  castling: { w: { kingSide: Boolean, queenSide: Boolean }, b: { ... } },
  enPassant: null,                // 或 { row, col }
  halfmove: 0,
  fullmove: 1,
}
```

### 题目分布（共 24 道）

| 主题       | 初级 | 中级 | 高级 | 小计 |
|------------|------|------|------|------|
| 残局 endgame   | 3    | 3    | 2    | 8    |
| 中局 middlegame| 2    | 3    | 2    | 7    |
| 战术 tactic    | 2    | 3    | 2    | 7    |
| 开局 opening   | 1    | 1    | 0    | 2    |
| **合计**   | 8    | 10   | 6    | **24** |

现有 6 道题迁移并补充 `difficulty`、`theme` 字段，新增 18 道。

## 弹窗 UI

### 触发方式
Puzzle 模式下，"换一题"按钮改为 **"选题"**，点击打开题目选择弹窗。

### 弹窗布局

```
┌─────────────────────────────────────────┐
│  选择题目                          [×]  │
│  ─────────────────────────────────────  │
│  难度: [全部] [初级] [中级] [高级]       │
│  主题: [全部] [残局] [中局] [战术][开局] │
│  ─────────────────────────────────────  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 车王残局  │ │ 后残局   │ │ 通路兵   │ │
│  │ 残局·初级 │ │ 残局·中级│ │ 残局·初级│ │
│  └──────────┘ └──────────┘ └──────────┘ │
│  ┌──────────┐ ...                        │
│  │(当前题目) │  ← 高亮正在进行的题       │
│  └──────────┘                            │
└─────────────────────────────────────────┘
```

### 交互规则
- 筛选器点击即时过滤，无需确认按钮
- 难度/主题可各选一个，也可保持"全部"
- 点击题目卡片直接加载，弹窗自动关闭
- 当前题目卡片有高亮边框
- 无匹配结果时显示"暂无符合条件的题目"

## 文件改动范围

### 新增
- `puzzles.js` — 24 道题目数据，暴露全局 `PUZZLES`

### 修改
- `index.html`
  - 在 `app.js` 之前引入 `<script src="./puzzles.js">`
  - 添加题目选择弹窗 HTML 结构（筛选器 + 题目网格容器）
  - "换一题"按钮 id 改为 `select-puzzle-btn`，文字改为"选题"

- `app.js`
  - 删除内联 `PUZZLES` 数组
  - 将 `nextPuzzleBtn` 改为 `selectPuzzleBtn`，绑定弹窗开关
  - 新增筛选状态（`puzzleFilter.difficulty`、`puzzleFilter.theme`）
  - 新增弹窗渲染函数（过滤 + 生成卡片 + 高亮当前题）
  - 新增筛选器点击事件处理
  - 估计新增约 80-100 行

- `styles.css`
  - 弹窗遮罩、弹窗卡片、关闭按钮样式
  - 筛选器 pill 按钮样式（选中态）
  - 题目卡片网格样式（hover、当前高亮）
  - 估计新增约 60-80 行

### 不动
- 核心棋局逻辑（走子、AI、复盘、悔棋）
- `server.js`、WebSocket 联机模式
- 人机、双人模式的任何逻辑

---

## 验收标准

1. Puzzle 模式下点击"选题"可打开弹窗
2. 难度/主题筛选器可单独或组合使用，列表即时更新
3. 无匹配时显示空状态提示
4. 点击题目卡片加载对应局面，弹窗关闭
5. 当前题目在弹窗中高亮
6. 24 道题目数据完整（id 唯一、board 合法、difficulty/theme 字段齐全）
7. 其他模式（人机、双人、联机）功能不受影响
